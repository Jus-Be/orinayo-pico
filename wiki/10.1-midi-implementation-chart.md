# MIDI Implementation Chart

> **Relevant source files**
> * [main.c](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c)
> * [pico_bluetooth.c](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c)

This page provides a complete reference of all MIDI messages transmitted and received by the Orinayo system. It documents the implementation of MIDI 1.0 standard messages, including Channel Voice Messages, System Real-Time Messages, and manufacturer-specific System Exclusive (SysEx) commands for supported hardware synthesizers.

For information about the dual USB/UART output architecture, see [6.1 Dual Output Architecture](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/6.1 Dual Output Architecture)

 For synthesizer-specific control commands, see [6.3 Synthesizer Control](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/6.3 Synthesizer Control)

---

## Overview

The Orinayo system implements a comprehensive MIDI output solution supporting:

* Standard Channel Voice Messages (Note On/Off, CC, PC)
* System Real-Time Messages (Clock, Start, Stop)
* Manufacturer-specific SysEx commands for Yamaha, Ketron, and Roland devices
* Simultaneous USB MIDI and UART MIDI output at 31,250 baud
* External MIDI clock synchronization for tempo-locked operation

All MIDI output is transmitted through the `midi_n_stream_write` function, which acts as the unified gateway for both physical and virtual interfaces.

## MIDI Messages Transmitted

### Channel Voice Messages

#### Note On (0x9n)

| **Usage Context** | **Channel** | **Note Range** | **Velocity** | **Function** |
| --- | --- | --- | --- | --- |
| Guitar/Lead | 1 (0x90) | Variable | 25-127 | Chord notes and melodies |
| Chord Track | 4 (0x93) | 36-47 (C1-B2) | 100-127 | Arranger chord voicing |
| Bass Track | 8 (0x97) | Variable | 100-127 | SeqTrak bass arpeggio |
| Chord Synth | 10 (0x99) | 36-47 (C1-B2) | 100-127 | SeqTrak DX synth |
| Drums | 10 (0x99) | 36-64 | 30-127 | GM drum kit notes |
| Sampler | 11 (0x9A) | 60-65 | 127 | SeqTrak chord samples |

**Note Assignment (GM Drum Map):**

* 36: Bass Drum
* 38: Snare Drum
* 42: Closed Hi-Hat
* 46: Open Hi-Hat
* 49: Crash Cymbal
* 56: Cowbell (metronome click)

#### Note Off (0x8n)

Transmitted for all Note On messages when notes are released. Velocity is typically 0.

**Sources:** [pico_bluetooth.c L189](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c#L189-L189)

#### Program Change (0xCn)

| **Channel** | **Range** | **Usage** |
| --- | --- | --- |
| 1 (0xC0) | 0-127 | Guitar patch selection (PC 26=Jazz default) |
| 4 (0xC3) | 0-127 | Pad voice selection |
| 1 (0xC0) | 0-15 | SeqTrak project selection |

**Sources:** [pico_bluetooth.c L119](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c#L119-L119)

#### Control Change (0xBn)

**Standard Controllers:**

| **CC#** | **Channel** | **Function** | **Value Range** | **Description** |
| --- | --- | --- | --- | --- |
| 0 | 1 | Bank MSB | 0-127 | Bank select for PC |
| 7 | 1, 4 | Volume | 0-127 | Channel volume |
| 32 | 1 | Bank LSB | 0-127 | Bank select LSB |
| 64 | 1 | Sustain | 0-127 | Sustain pedal (on/off) |
| 91 | 1 | Reverb Level | 0-127 | Effect send level |
| 92 | 4 | Scene Select | 0-127 | MODX scene switching |

### System Real-Time Messages

#### MIDI Clock (0xF8)

Transmitted at 24 PPQN (pulses per quarter note) when acting as master. Received to synchronize the looper to an external source.

**Sources:** [pico_bluetooth.c L188](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c#L188-L188)

#### MIDI Start (0xFA) / Stop (0xFC)

Transmitted via `midi_start_stop()` to control external sequencers and the internal looper engine simultaneously.

**Sources:** [pico_bluetooth.c L188](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c#L188-L188)

### System Exclusive Messages

The system implements manufacturer-specific SysEx commands for multiple hardware synthesizers.

**Common MODX/SeqTrak Commands:**

* **Tempo:** `midi_modx_tempo()` [pico_bluetooth.c L181](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c#L181-L181)  `midi_seqtrak_tempo()` [pico_bluetooth.c L185](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c#L185-L185)

#### Roland Dream (M5Audio) SysEx

Used to set delay times relative to the current BPM.

* **Formula:** `(60000 / tempo / 128 / 4) % 128`
* **Source:** `dream_set_delay()` [pico_bluetooth.c L217](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c#L217-L217)

#### Arranger Control (Yamaha/Ketron)

* **Yamaha Arranger:** Controls sections (Intro, Main, Fill, Ending) via `midi_yamaha_arr()`. [pico_bluetooth.c L197](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c#L197-L197)

## Implementation Details

### UART Configuration

The Raspberry Pi Pico UART0 is configured specifically for MIDI baud rates.

* **Baud Rate:** 31,250 [main.c L66](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L66-L66)
* **Pins:** TX (GPIO 0), RX (GPIO 1) [main.c L67-L68](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L67-L68)
* **Initialization:** `uart_init(UART_ID, BAUD_RATE)` [main.c L220](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L220-L220)

### USB MIDI Class

Handled by the TinyUSB stack.

* **Task:** `tud_task()` must be called frequently in the main loop. [main.c L228](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L228-L228)
* **Connection State:** Monitored via `tud_mount_cb` and `tud_umount_cb` to update the onboard LED. [main.c L209-L218](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L209-L218)
