## Bluepad32 Configuration (sdkconfig.h)

The [sdkconfig.h L1-L12](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/sdkconfig.h#L1-L12)

 file configures the Bluepad32 controller library. This file controls the maximum number of simultaneous Bluetooth connections, security requirements, and logging verbosity.

### Configuration Macros

| Macro | Default Value | Valid Range | Description |
| --- | --- | --- | --- |
| `CONFIG_BLUEPAD32_PLATFORM_CUSTOM` | Defined | - | Enables custom platform implementation (Pico W) [sdkconfig.h L4](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/sdkconfig.h#L4-L4) |
| `CONFIG_TARGET_PICO_W` | Defined | - | Specifies target hardware is Raspberry Pi Pico W [sdkconfig.h L5](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/sdkconfig.h#L5-L5) |
| `CONFIG_BLUEPAD32_LOG_LEVEL` | `0` | 0-4 | Log verbosity: 0=INFO, 1=WARN, 2=ERROR [sdkconfig.h L6](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/sdkconfig.h#L6-L6) |
| `CONFIG_BLUEPAD32_MAX_DEVICES` | `1` | 1-4 | Maximum simultaneous Bluetooth devices [sdkconfig.h L8](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/sdkconfig.h#L8-L8) |
| `CONFIG_BLUEPAD32_MAX_ALLOWLIST` | `1` | 0-16 | Maximum devices in pairing allowlist [sdkconfig.h L9](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/sdkconfig.h#L9-L9) |
| `CONFIG_BLUEPAD32_GAP_SECURITY` | `1` | 0-3 | Security mode: 1=Encryption [sdkconfig.h L10](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/sdkconfig.h#L10-L10) |
| `CONFIG_BLUEPAD32_ENABLE_BLE_BY_DEFAULT` | `1` | 0-1 | Enable BLE scanning on startup [sdkconfig.h L11](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/sdkconfig.h#L11-L11) |

### Usage Guidelines

**Increasing MAX_DEVICES**: The default value of `1` [sdkconfig.h L8](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/sdkconfig.h#L8-L8)

 is optimized for single-controller use cases. Increasing this requires corresponding changes in [bluepad32/include/btstack_config.h L47](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/bluepad32/include/btstack_config.h#L47-L47)

 (`MAX_NR_HCI_CONNECTIONS`) to allocate sufficient connection slots.

**Security Levels**: The default `GAP_SECURITY=1` [sdkconfig.h L10](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/sdkconfig.h#L10-L10)

 enables encryption. For devices requiring stronger authentication, increase to `2` or `3`.

**BLE Mode**: The system requires `ENABLE_BLE_BY_DEFAULT=1` [sdkconfig.h L11](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/sdkconfig.h#L11-L11)

 to support Liberlive and Sonic Master MIDI devices.

**Sources:** [sdkconfig.h L1-L12](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/sdkconfig.h#L1-L12)

### Seqlock Infrastructure (seqlock.h)

The [seqlock.h L1-L105](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/seqlock.h#L1-L105)

 file provides a lightweight single-writer / multiple-reader lock.

* **Writer**: Uses `seqlock_write_begin` [seqlock.h L40](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/seqlock.h#L40-L40)  and `seqlock_write_end` [seqlock.h L45](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/seqlock.h#L45-L45)
* **Reader**: Uses `SEQLOCK_TRY_READ` [seqlock.h L59](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/seqlock.h#L59-L59)  macro to obtain a consistent snapshot of shared data (e.g., gamepad state).
