## Bluetooth Input System

The Bluetooth Input System handles device discovery, connection management, HID report parsing, and translation of controller button states into musical commands.

### Key Components

| Component | File | Primary Responsibilities |
| --- | --- | --- |
| HID-to-MIDI Gateway | `pico_bluetooth.c` | Button state parsing, chord generation, operational mode selection |
| BLE MIDI Client | `ble_midi_client.c` | Connection and GATT handling for specialized MIDI controllers |
| Controller Abstraction | Bluepad32 library | Standardized gamepad/controller interface via BTstack |

### Button State Management

The system maintains global state variables for all controller inputs:

* **Fret buttons**: `green`, `red`, `yellow`, `blue`, `orange` [pico_bluetooth.c L96-L100](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c#L96-L100)
* **Strum/D-Pad inputs**: `dpad_left`, `dpad_right`, `up`, `down` [pico_bluetooth.c L86-L106](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c#L86-L106)
* **Action buttons**: `mbut0` (start/stop), `mbut1` (style), `mbut2` (menu), `mbut3` (config) [pico_bluetooth.c L81-L84](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c#L81-L84)
* **Modifiers**: `pitch`, `song_key`, `start` [pico_bluetooth.c L102-L110](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c#L102-L110)

The system extracts button states from the controller structures provided by Bluepad32 and updates these global variables. The `gamepad_bluetooth_handle_data()` function [pico_bluetooth.c L199](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/pico_bluetooth.c#L199-L199)

 processes these state changes.

## Musical Processing Subsystem

The Musical Processing Subsystem implements the looper, ghost note generation, tap tempo detection, and note scheduler.

### Looper and Scheduler Interaction

The looper coordinates with the note scheduler to ensure timing precision. Notes generated by the step sequencer are not sent directly to the hardware; instead, they are queued via the `note_scheduler` to ensure timing precision and USB safety.

```mermaid
flowchart TD

Perform["looper_perform_step()"]
Schedule["note_scheduler_schedule_note()"]
Queue["Scheduled Notes Queue"]
Dispatch["note_scheduler_dispatch_pending()"]
Output["midi_n_stream_write()"]

Perform --> Schedule
Schedule --> Queue
Queue --> Dispatch
Dispatch --> Output
```

The `note_scheduler_dispatch_pending()` function is called within the `main` loop [main.c L244](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L244-L244)

 to flush pending MIDI messages.

### Dual Output Implementation

The `midi_n_stream_write()` function [main.c L165](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L165-L165)

 implements the dual-output strategy, ensuring that every MIDI message is sent to both the TinyUSB stack and the hardware UART. The UART is initialized to the standard MIDI baud rate of 31250 [main.c L67](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L67-L67)

## Infrastructure Layer

The Infrastructure Layer provides supporting functionality for timing, persistence, and hardware abstraction.

### Asynchronous Timer System

The `async_timer.c` module [async_timer.c L1-L34](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/async_timer.c#L1-L34)

 provides a unified asynchronous context abstraction. This is critical for the Pico W architecture where the Bluetooth stack (CYW43) requires specific context handling.

* **Pico W with CYW43 Bluetooth**: Returns `cyw43_arch_async_context()` to use the WiFi chip's built-in async context [async_timer.c L28-L29](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/async_timer.c#L28-L29)
* **Other Pico boards**: Initializes and returns a `async_context_threadsafe_background_t` [async_timer.c L14-L32](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/async_timer.c#L14-L32)

The system utilizes flash storage via `storage.c` to persist looper patterns and user preferences. When `preferences_changed` is flagged [main.c L100](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L100-L100)
