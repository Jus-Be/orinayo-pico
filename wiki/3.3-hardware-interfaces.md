## Purpose and Scope

This page documents the hardware interfaces available on the Raspberry Pi Pico 2 W platform and how the Orinayo system utilizes them. It covers the physical layer configuration, initialization sequences, and interface-specific functions for USB, UART, Bluetooth, and GPIO.

For information about higher-level system architecture, see [System Components](./3.1-system-components.md). For USB MIDI descriptor configuration, see [USB MIDI Configuration](./6.2-usb-midi-configuration.md). For details on MIDI output logic, see [Dual Output Architecture](./6.1-dual-output-architecture.md).

## Hardware Platform

The Orinayo system runs on the **Raspberry Pi Pico 2 W**, which provides the following hardware capabilities:

| Component | Specification | Usage in Orinayo |
| --- | --- | --- |
| Microcontroller | RP2350 dual-core ARM Cortex-M33 | Core 0: Application logic, USB, UARTCore 1: Available for future use |
| Wireless Chip | CYW43 (Bluetooth + WiFi) | Bluetooth Classic + BLEWiFi unused |
| Flash Storage | 4MB | Firmware + pattern storage |
| GPIO Pins | 26 digital I/O | Button input, LED output, UART |
| USB | USB 1.1 Device | MIDI device interface |
| UART | 2 hardware UARTs | UART0 for MIDI output, UART1 for M5Audio |

The system is configured for the `rp2350-arm-s` platform and `pico2_w` board in the build system.

## USB Interface

The Orinayo system operates in a **USB dual-role mode**, acting as both a USB device and a USB host.

* **USB Device:** The native USB port on the Pico W (Port 0) functions as a USB MIDI device, allowing connection to a computer or other USB host.
* **USB Host:** A PIO-based USB host (Port 1) is implemented to connect to external USB MIDI controllers.

This dual-role functionality is enabled by defining `CFG_TUD_ENABLED` and `CFG_TUH_ENABLED` as `1` in `tusb_config.h`. [tusb_config.h L50-L51](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/tusb_config.h#L50-L51)

### USB Device Configuration

The Orinayo system presents itself as a **USB MIDI device** using the TinyUSB library. The USB configuration is statically defined in descriptors.

**Key USB Identifiers:**

| Parameter | Value | Description |
| --- | --- | --- |
| Vendor ID | `0xCafe` | Fixed identifier |
| Product ID | `USB_PID` | Dynamic, bit-mapped based on enabled interfaces |
| Device Class | `0x00` | Composite device |
| USB Version | `0x0200` | USB 2.0 |
| Max Packet Size | `64` (Full Speed)`512` (High Speed) | MIDI endpoint size |

The Product ID is automatically calculated using a bitmap of enabled TinyUSB interfaces. In `tusb_config.h`, `CFG_TUD_MIDI` is set to 1. [tusb_config.h L90](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/tusb_config.h#L90-L90)

```
// From usb_descriptors.c:34-36#define _PID_MAP(itf, n)  ( (CFG_TUD_##itf) << (n) )#define USB_PID           (0x4000 | _PID_MAP(CDC, 0) | _PID_MAP(MSC, 1) | _PID_MAP(HID, 2) | \                           _PID_MAP(MIDI, 3) | _PID_MAP(VENDOR, 4) )
```

### USB Host Configuration (PIO USB)

The Pico W's second Programmable I/O (PIO) block is utilized to implement a USB host controller. This allows the Orinayo system to connect to and receive MIDI data from external USB MIDI devices.

**Key Configuration Macros:**

* `BOARD_TUH_RHPORT = 1`: Designates PIO USB as root hub port 1 for host functionality. [tusb_config.h L41](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/tusb_config.h#L41-L41)
* `CFG_TUSB_RHPORT1_MODE = OPT_MODE_HOST`: Configures root hub port 1 for host mode. [tusb_config.h L43](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/tusb_config.h#L43-L43)
* `CFG_TUH_RPI_PIO_USB = 1`: Enables PIO-USB as the host controller. [tusb_config.h L103](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/tusb_config.h#L103-L103)
* `PICO_DEFAULT_PIO_USB_DP_PIN = 17`: Specifies GPIO 17 as the D+ pin for the PIO USB host. [CMakeLists.txt L52](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L52-L52)

The `pio_usb_configuration_t` struct is used to pass PIO-specific settings to the TinyUSB host stack during initialization. [tinyusb/hw/bsp/rp2040/family.c L188-L190](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/tinyusb/hw/bsp/rp2040/family.c#L188-L190)

### UART Initialization and Transmission

```mermaid
sequenceDiagram
  participant Application Logic
  participant m5audio.c
  participant UART1 Hardware

  Application Logic->>m5audio.c: m5audio_init()
  m5audio.c->>UART1 Hardware: uart_init(uart1, 9600)
  m5audio.c->>UART1 Hardware: gpio_set_function(4, GPIO_FUNC_UART)
  Application Logic->>m5audio.c: m5audio_play_track(track)
  m5audio.c->>m5audio.c: audioplayer_send(0x04, data, 3)
  loop [for each byte in frame]
    m5audio.c->>UART1 Hardware: uart_is_writable(uart1)
    m5audio.c->>UART1 Hardware: uart_putc(uart1, byte)
  end
```

The `audioplayer_send` function builds a frame consisting of a command, its complement, length, data, and a checksum before transmitting over UART1.

## Bluetooth Interface

### CYW43 Wireless Chip Integration

The CYW43 chip on the Pico 2 W provides Bluetooth Classic and BLE capabilities. The system uses **threadsafe background operations** to allow Bluetooth processing to occur asynchronously.

**Build Configuration:**

| Library | Purpose |
| --- | --- |
| `pico_cyw43_arch_threadsafe_background` | Threadsafe async Bluetooth operations |
| `pico_btstack_classic` | Bluetooth Classic stack |
| `pico_btstack_ble` | Bluetooth Low Energy stack |
| `bluepad32` | Game controller support library |
