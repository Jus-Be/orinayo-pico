This document covers the custom BLE GATT protocol implementation for Liberlive and Sonic Master MIDI controller devices. These specialized devices communicate using proprietary GATT characteristics rather than standard HID reports.

For generic gamepad/guitar controller input handling via standard HID, see [4.2 HID to MIDI Translation](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/4.2 HID to MIDI Translation)

 For BLE device connection management and pairing, see [4.1 BLE Device Management](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/4.1 BLE Device Management)

## Overview

The system supports two specialized BLE MIDI devices that use custom GATT services:

| Device Name | Advertising Name | Service UUID | Purpose |
| --- | --- | --- | --- |
| Liberlive | "Liber" (prefix) | `000000ff-0000-1000-8000-00805f9b34fb` | Chord controller with drum/chord pads |
| Sonic Master | "Pocket Master BLE" | `03b80e5a-ede8-4b33-a751-6ce34ec4c700` | Portable MIDI controller |

These devices bypass the standard Bluepad32 HID processing and use a custom GATT notification-based protocol implemented in `bluepad32/bt/uni_bt_le.c`.

**Sources:** [bluepad32/bt/uni_bt_le.c L78-L79](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/bluepad32/bt/uni_bt_le.c#L78-L79)

### GATT Service Discovery

Upon BLE connection, the system initiates GATT service and characteristic discovery via the `handle_gatt_client_event` callback.

```mermaid
stateDiagram-v2
    [*] --> ConnectionComplete : "HCI_SUBEVENT_LE_CONNECTION_COMPLETE"
    ConnectionComplete --> DiscoverService : "gatt_client_discover_primary_services_by_uuid128"
    DiscoverService --> ServiceFound : "GATT_EVENT_SERVICE_QUERY_RESULTquery_state = 0"
    ServiceFound --> DiscoverChar : "gatt_client_discover_characteristics_for_service_by_uuid128"
    DiscoverChar --> CharFound : "GATT_EVENT_CHARACTERISTIC_QUERY_RESULTquery_state = 1"
    CharFound --> ConfigWrite : "GATT_EVENT_QUERY_COMPLETE"
    ConfigWrite --> ListenNotify : "gatt_client_listen_for_characteristic_value_updates"
    ListenNotify --> Ready : "query_state = 2-4"
    Ready --> [*] : "query_state = 2-4"
```

The discovery process uses 128-bit UUIDs specific to each device:

| Device | Service UUID (Little-Endian Bytes) | Characteristic UUID |
| --- | --- | --- |
| Liberlive | `{0x00,0x00,0x00,0xff,...,0xFB}` | `{0x00,0x00,0xff,0x03,...,0xfb}` |
| Sonic Master | `{0x03,0xB8,0x0E,0x5A,...,0x00}` | `{0x77,0x72,0xE5,0xDB,...,0xF3}` |

**Sources:** [bluepad32/bt/uni_bt_le.c L1220-L1256](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/bluepad32/bt/uni_bt_le.c#L1220-L1256)

## Liberlive Configuration

During the GATT setup phase (`query_state` 1-4), the system writes configuration data to the Liberlive device using `gatt_client_write_value_of_characteristic`.

### Chord Key Mapping

The chord mapping configuration is a 26-byte array that defines how the device interprets chord inputs:

```
static uint8_t chord_mappings[26] = {    177, 30, 31, 21, 0, 128, 147, 117, 5, 85, 81, 113,     160, 145, 112, 0, 80, 33, 65, 176, 144, 112, 0, 48, 32, 64};
```

This configuration is written to the device characteristic to initialize its internal mapping engine.

**Sources:** [bluepad32/bt/uni_bt_le.c L828-L830](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/bluepad32/bt/uni_bt_le.c#L828-L830)

### Optional Configuration Commands

The code includes commented-out configuration options for fine-tuning the Liberlive hardware:

| Configuration | Byte Array | Purpose |
| --- | --- | --- |
| Drum Preview | `{177, 30, 38, 1, 0, 1}` | Enable/disable drum sound preview |
| Chord Preview | `{177, 30, 37, 1, 0, 1}` | Enable/disable chord sound preview |
| Chord Tip | `{177, 30, 12, 3, 0, 2, 2, 1}` | Chord display tip settings |

| Set Key | `{177, 30, 24, 1, 0, 0}` | Set musical key (C/D/E/etc) |

## Input Decoding

### Paddle State Detection

The paddle state (byte 5) determines the input mode and velocity calculation. Strum velocity is derived from bytes 9 and 10.

```mermaid
flowchart TD

CheckPaddle["event_data[5]<br>value?"]
Neutral["0: Neutral<br>ll_cannot_fire = false"]
PaddleA["12: Paddle A<br>Check byte[9] velocity"]
PaddleB["3: Paddle B<br>Check byte[10] velocity"]
ConfigMode["64: Config/Stop mode<br>Guitar play mode selection"]
VelUp["< 48: Up strum<br>applied_velocity = (50-value)/50"]
VelDown["> 58: Down strum<br>applied_velocity = value/50"]

CheckPaddle --> Neutral
CheckPaddle --> PaddleA
CheckPaddle --> PaddleB
CheckPaddle --> ConfigMode
PaddleA --> VelUp
PaddleA --> VelDown
```

The paddle velocity bytes map to the `applied_velocity` variable. Thresholds create a dead zone for the neutral position.

**Sources:** [bluepad32/bt/uni_bt_le.c L912-L916](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/bluepad32/bt/uni_bt_le.c#L912-L916)

## Fire Control Mechanism

The system implements a "fire control" mechanism using `ll_have_fired` and `ll_cannot_fire` to prevent duplicate events from a single paddle motion.

```mermaid
stateDiagram-v2
    [*] --> Idle : "ll_have_fired = falsell_cannot_fire = false"
    Idle --> Fired : "Paddle strummedll_have_fired = true"
    Fired --> CannotFire : "ll_cannot_fire = true"
    CannotFire --> Neutral : "event_data[5] == 0"
    Neutral --> Reset : "ll_have_fired = falsell_cannot_fire = false"
    Reset --> Idle : "ll_cannot_fire = true"
```

## Disconnection Handling

When a specialized device disconnects, the system clears flags in `uni_bt_le_on_hci_disconnection_complete`.

```
void uni_bt_le_on_hci_disconnection_complete(hci_con_handle_t con_handle) {    liberlive_enabled = false;    sonic_master_enabled = false;    resume_scanning_hint();}
```

**Sources:** [bluepad32/bt/uni_bt_le.c L1392-L1400](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/bluepad32/bt/uni_bt_le.c#L1392-L1400)

## Sending MIDI to Specialized Devices

The system can send MIDI data back to these devices using `send_ble_midi`, which performs a GATT write-without-response.

```
void send_ble_midi(uint8_t* midi_data, int len) {    gatt_client_write_value_of_characteristic_without_response(        connection_handle,         server_characteristic.value_handle,         len,         midi_data    );}
```
