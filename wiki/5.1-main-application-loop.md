## Purpose and Scope

This page documents the main application entry point and event loop implemented in [main.c](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c)

 It covers the system initialization sequence, the main event processing loop, coordination of subsystems, and the critical dual-output MIDI transmission mechanism. For details on the musical processing components coordinated by this loop, see [Step Sequencer](./5.2-step-sequencer.md), [Note Scheduler](./5.5-note-scheduler.md), and [Ghost Note Generation](./5.4-ghost-note-generation.md). For Bluetooth initialization and device management, see [BLE Device Management](./4.1-ble-device-management.md).

## Main Event Loop

The main event loop is a tight polling loop that coordinates USB processing, incoming MIDI handling, note dispatch, and system persistence.

### Event Loop Structure

```

```

Sources: [main.c L227-L250](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L227-L250)

### Loop Components

| Component | Function | Frequency | Purpose |
| --- | --- | --- | --- |
| `tud_task()` | USB device task | Every iteration | Poll USB events and endpoints |
| MIDI input | `tud_midi_available()` | When data present | Read incoming USB MIDI packets and forward to USB Host if `device_addr` is set |
| Note dispatch | `note_scheduler_dispatch_pending()` | Every iteration | Output scheduled notes with precise timing |
| Storage | `storage_store_preferences()` | On change | Persist settings to flash and reboot |

Sources: [main.c L227-L250](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L227-L250)

## External Variables and Global State

The main file coordinates several external variables that reflect the current operational state, modified primarily by the Bluetooth input subsystem:

| Variable | Type | Purpose |
| --- | --- | --- |
| `style_started` | `bool` | Indicates if sequencer/arranger is running |
| `enable_midi_drums` | `bool` | Looper/Drums mode toggle |
| `enable_seqtrak` | `bool` | Yamaha SeqTrak mode toggle |
| `enable_modx` | `bool` | Yamaha MODX mode toggle |
| `preferences_changed` | `bool` | Trigger for flash storage and reboot |
