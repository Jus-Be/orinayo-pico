## Purpose and Scope

This page documents the USB MIDI device configuration for the Orinayo system, including USB descriptors, TinyUSB library settings, and device enumeration. This covers how the Raspberry Pi Pico 2 W presents itself as a USB MIDI device to host computers.

For information about how MIDI data is transmitted over USB, see [Dual Output Architecture](./6.1-dual-output-architecture.md). For MIDI message content and synthesizer-specific commands, see [Synthesizer Control](./6.3-synthesizer-control.md).

## USB Device Identification

The Orinayo device uses a fixed Vendor ID (VID) with a dynamically generated Product ID (PID) based on enabled USB classes.

### Vendor and Product IDs

```yaml
VID: 0xCafe
PID: Dynamic (base 0x4000)
```

The Product ID is computed using a bitmap that encodes which USB device classes are enabled:

```
[MSB]  MIDI | HID | MSC | CDC  [LSB]
```

The PID calculation is defined in [usb_descriptors.c L34-L36](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L34-L36)

:

```
#define _PID_MAP(itf, n)  ( (CFG_TUD_##itf) << (n) )#define USB_PID           (0x4000 | _PID_MAP(CDC, 0) | _PID_MAP(MSC, 1) | _PID_MAP(HID, 2) | \                           _PID_MAP(MIDI, 3) | _PID_MAP(VENDOR, 4) )
```

In the current configuration with only MIDI enabled (`CFG_TUD_MIDI = 1`), the computed PID is `0x4008` (0x4000 | (1 << 3)).

### Device Descriptor

The device descriptor is defined in [usb_descriptors.c L41-L60](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L41-L60)

 and contains:

| Field | Value | Description |
| --- | --- | --- |
| `bcdUSB` | 0x0200 | USB 2.0 specification |
| `bDeviceClass` | 0x00 | Defined at interface level |
| `bDeviceSubClass` | 0x00 | Defined at interface level |
| `bDeviceProtocol` | 0x00 | Defined at interface level |
| `bMaxPacketSize0` | 64 | Control endpoint size |
| `idVendor` | 0xCafe | Fixed vendor ID |
| `idProduct` | Dynamic | Computed from enabled classes |
| `bcdDevice` | 0x0100 | Device version 1.0 |
| `iManufacturer` | 1 | String descriptor index |
| `iProduct` | 2 | String descriptor index |
| `iSerialNumber` | 3 | String descriptor index |
| `bNumConfigurations` | 1 | Single configuration |

The device descriptor is returned via the `tud_descriptor_device_cb()` callback function at [usb_descriptors.c L64-L67](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L64-L67)

**Sources:** [usb_descriptors.c L28-L67](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L28-L67)

## TinyUSB Configuration

The TinyUSB stack is configured through [tusb_config.h](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/tusb_config.h)

 which defines device class enablement and buffer sizes.

### Device Class Configuration

Only the MIDI device class is enabled in [tusb_config.h L80-L84](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/tusb_config.h#L80-L84)

:

```
#define CFG_TUD_CDC             0#define CFG_TUD_MSC             0#define CFG_TUD_HID             0 #define CFG_TUD_MIDI            1 #define CFG_TUD_VENDOR          0
```

### Buffer Configuration

MIDI transmit and receive FIFO sizes are configured to automatically scale based on USB speed at [tusb_config.h L87-L88](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/tusb_config.h#L87-L88)

:

```
#define CFG_TUD_MIDI_RX_BUFSIZE   (TUD_OPT_HIGH_SPEED ? 512 : 64)#define CFG_TUD_MIDI_TX_BUFSIZE   (TUD_OPT_HIGH_SPEED ? 512 : 64)
```

### Operating System Configuration

The stack is configured to use the Pico SDK's operating system abstraction at [tusb_config.h L45](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/tusb_config.h#L45-L45)

:

```
#define CFG_TUSB_OS                 OPT_OS_PICO
```

### Root Hub Port Mode

The device operates in device mode on root hub port 0, configured at [tusb_config.h L47](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/tusb_config.h#L47-L47)

:

```
#define CFG_TUSB_RHPORT0_MODE     OPT_MODE_DEVICE
```

## MIDI Interface Configuration

The USB configuration includes two interfaces for MIDI: a standard Audio Control interface and a MIDI Streaming interface.

### Interface Enumeration

Defined in [usb_descriptors.c L74-L79](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L74-L79)

:

```
enum{  ITF_NUM_MIDI = 0,  ITF_NUM_MIDI_STREAMING,  ITF_NUM_TOTAL};
```

### Endpoint Assignment

The MIDI endpoint number is platform-dependent, defined at [usb_descriptors.c L83-L89](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L83-L89)

:

* LPC 17xx/40xx: `EPNUM_MIDI = 0x02` (fixed by hardware)
* Other platforms (including RP2350/RP2040): `EPNUM_MIDI = 0x01`

The configuration uses bidirectional endpoints:

* OUT endpoint: `EPNUM_MIDI` (host to device)
* IN endpoint: `0x80 | EPNUM_MIDI` (device to host)

### Configuration Descriptor Structure

The total configuration length is computed at [usb_descriptors.c L81](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L81-L81)

:

```
#define CONFIG_TOTAL_LEN  (TUD_CONFIG_DESC_LEN + TUD_MIDI_DESC_LEN)
```

## USB Speed Support

The device supports both full-speed (FS) and high-speed (HS) USB operation with different endpoint sizes.

### Full-Speed Configuration

Defined at [usb_descriptors.c L91-L98](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L91-L98)

:

* **Endpoint Size**: 64 bytes
* **Macro**: `TUD_MIDI_DESCRIPTOR(ITF_NUM_MIDI, 0, EPNUM_MIDI, 0x80 | EPNUM_MIDI, 64)`

### High-Speed Configuration

Conditionally compiled when `TUD_OPT_HIGH_SPEED` is defined at [usb_descriptors.c L100-L109](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L100-L109)

:

* **Endpoint Size**: 512 bytes
* **Macro**: `TUD_MIDI_DESCRIPTOR(ITF_NUM_MIDI, 0, EPNUM_MIDI, 0x80 | EPNUM_MIDI, 512)`

### Speed Detection

The `tud_descriptor_configuration_cb()` callback at [usb_descriptors.c L114-L124](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L114-L124)

 selects the appropriate descriptor based on the negotiated speed:

```javascript
uint8_t const * tud_descriptor_configuration_cb(uint8_t index){  (void) index;#if TUD_OPT_HIGH_SPEED  return (tud_speed_get() == TUSB_SPEED_HIGH) ?  desc_hs_configuration : desc_fs_configuration;#else  return desc_fs_configuration;#endif}
```

## String Descriptors

The device provides string descriptors for identification, defined in the `string_desc_arr` array at [usb_descriptors.c L131-L139](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L131-L139)

### String Descriptor Table

| Index | Purpose | Value |
| --- | --- | --- |
| 0 | Language ID | 0x0409 (English - US) |
| 1 | Manufacturer | "Inspired Futures Ltd" |
| 2 | Product | "Orinayo Device" |
| 3 | Serial Number | "123456" |
| 4 | CDC Interface | "Orinayo CDC" |
| 5 | MIDI Interface | "Orinayo MIDI" |

### String Descriptor Callback

The `tud_descriptor_string_cb()` function at [usb_descriptors.c L145-L179](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L145-L179)

 handles string descriptor requests:

1. Validates the requested string index against `string_desc_arr` [usb_descriptors.c L160](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L160-L160)
2. Converts ASCII strings to UTF-16 encoding [usb_descriptors.c L169-L172](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L169-L172)
3. Limits string length to 31 characters maximum [usb_descriptors.c L166-L167](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L166-L167)
4. Formats the descriptor with proper length and type fields in `_desc_str` [usb_descriptors.c L176](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L176-L176)

The function uses a static buffer `_desc_str[32]` declared at [usb_descriptors.c L141](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L141-L141)

 to store the formatted descriptor.

## Device Enumeration Process

When the Orinayo device is connected to a USB host, the following enumeration sequence occurs:

1. **Device Detection**: Host detects device connection and issues a USB reset.
2. **Device Descriptor Request**: Host requests device descriptor via `tud_descriptor_device_cb()` [usb_descriptors.c L64-L67](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L64-L67)
3. **Address Assignment**: Host assigns a unique USB address to the device.
4. **Configuration Descriptor Request**: Host requests configuration descriptor via `tud_descriptor_configuration_cb()` [usb_descriptors.c L114-L124](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L114-L124)
5. **String Descriptor Requests**: Host may request manufacturer, product, and serial number strings via `tud_descriptor_string_cb()` [usb_descriptors.c L145-L179](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L145-L179)
6. **Configuration Selection**: Host selects configuration 1.
7. **MIDI Interface Setup**: Host initializes the MIDI Audio Control and MIDI Streaming interfaces defined in `desc_fs_configuration` [usb_descriptors.c L91-L98](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L91-L98)
8. **Endpoint Initialization**: Host configures bulk IN and OUT endpoints using `EPNUM_MIDI`.
9. **Device Ready**: Device is enumerated and ready to transmit/receive MIDI messages.

**Sources:** [usb_descriptors.c L64-L179](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/usb_descriptors.c#L64-L179)
