This page documents the synthesizer-specific control systems that send SysEx (System Exclusive) and other MIDI messages to configure and control various hardware synthesizers and arrangers. The system supports Yamaha MODX, Yamaha SeqTrak, Ketron arrangers, Yamaha PSR arrangers, and Roland Dream effects processors.

For general MIDI output architecture, see [Dual Output Architecture](./6.1-dual-output-architecture.md). For USB and UART configuration, see [USB MIDI Configuration](./6.2-usb-midi-configuration.md).

## Overview

The Orinayo system implements device-specific control protocols for multiple synthesizer families. These controls are triggered by button combinations on the guitar controller and are mode-dependent (controlled by `enable_modx`, `enable_seqtrak`, and `enable_arranger_mode` flags).

All synthesizer control commands are sent through `midi_n_stream_write()`, which simultaneously transmits to USB and UART MIDI outputs.

## Yamaha MODX Control

The MODX synthesizer is controlled via Yamaha's XG/MODX-specific SysEx protocol. The system can configure arpeggiator parameters, tempo, key, and scene selection.

### MODX SysEx Message Structure

All MODX SysEx messages follow this format:
`F0 43 10 7F 1C 0D [address bytes] [data bytes] F7`

| Byte Position | Value | Description |
| --- | --- | --- |
| 0 | `0xF0` | SysEx start |
| 1 | `0x43` | Yamaha manufacturer ID |
| 2 | `0x10` | Device number |
| 3 | `0x7F` | Model ID |
| 4 | `0x1C` | Parameter change |
| 5 | `0x0D` | Address high |
| 6-9 | Variable | Address and data bytes |
| Last | `0xF7` | SysEx end |

### MODX Arpeggiator and Tempo Control

The `midi_modx_arp()` function [main.c L271-L289](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L271-L289)

 enables or disables the MODX arpeggiator. The octave range is controlled by `midi_modx_arp_octave()` [main.c L251-L269](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L251-L269)

 where the value is offset by `0x40` in the SysEx message.

Tempo is set using a two-byte value (MSB/LSB split) in `midi_modx_tempo()` [main.c L333-L352](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L333-L352)

 The tempo value is split into MSB (`tempo / 128`) and LSB (`tempo % 128`).

### SP-404mkII Sample Management

The `tools/ketron2sp404-1.cmd`, `tools/ketron2sp404-2.cmd`, and `tools/ketron2sp404-3.cmd` scripts demonstrate how WAV files are organized and copied for use with the SP-404mkII. These scripts map specific WAV files (e.g., `ARRA.wav`, `MAJ_C.wav`) to numbered sample slots (e.g., `1-01-120.wav`, `2-01-120.wav`) on different banks. This suggests that the `sp404_trigger_loop()` function would send MIDI Note On messages corresponding to these sample slots.

For example, `tools/ketron2sp404-1.cmd` copies `ARRA.wav` to `.\01\SAMPLE\1-01-085.wav`, indicating a sample at bank 1, pad 1.

Sources: [tools/ketron2sp404-1.cmd L3-L138](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/tools/ketron2sp404-1.cmd#L3-L138)

### Roland RC-600 Sample Management

The `tools/ketron2rc600-1.cmd` and `tools/ketron2rc600-2.cmd` scripts use `ffmpeg` to concatenate WAV files for the Roland RC-600. This process combines two input WAV files (e.g., `IN/1A/MAJ_C.wav` and `IN/1B/MAJ_C.wav`) into a single output WAV file (e.g., `WAVE/001_1/MAJ_C.wav`). The `tools/RC600-1.txt` and `tools/RC600-2.txt` files then reference these generated WAV files for import/export, indicating how the RC-600 manages its samples.

## Roland Dream Delay Control

The Roland Dream effect processor's delay time is synchronized via `dream_set_delay()` [main.c L210-L229](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/main.c#L210-L229)

 It calculates a rate based on tempo: `rate = (60000 / tempo / 128 / 4) % 128`.

## Mode-Based Control Routing

The synthesizer control functions check mode flags before executing. Mode selection is handled in the Bluetooth HID processing layer.

| Fret Combo | Mode Toggled | Flag |
| --- | --- | --- |
