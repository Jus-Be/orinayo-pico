The Status Display system provides a real-time text-based user interface over UART that visualizes the looper's state, track patterns, and playback position. This console output enables users to monitor the step sequencer without external hardware displays.

For information about button input that controls these patterns, see [Button Input](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/Button Input)

## Purpose and Architecture

The status display system renders the looper's current state to a UART serial console at 115200 baud, refreshing on every step tick. The implementation resides in [display.c L1-L111](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L1-L111)

 and is invoked by the looper's main state machine.

### System Entry Point

The looper calls `display_update_looper_status()` on every step to refresh the display:


## Display Format

The display renders a grid showing 14 drum tracks across 32 sixteenth-note steps (2 bars). Each refresh cycle outputs the complete status in a standardized format.

### Status Header

```
#state PLAYING
#bpm 120
#grid                  1   2   3   4   5   6   7   8
```

The header shows:

* **State**: Current looper state (WAITING, PLAYING, RECORDING, TAP TEMPO, MUTE) [display.c L78-L98](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L78-L98)
* **BPM**: Current tempo [display.c L101](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L101-L101)
* **Grid**: Bar numbers (8 bars = 32 steps) [display.c L103](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L103-L103)

**Sources:** [display.c L78-L103](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L78-L103)

### Track Representation

Each track is rendered as a row with the following format:

```
#track N > Track_Name  ________________________________
#track N _ Track_Name  *___.*__.+_______________________
```

| Element | Symbol | Meaning |
| --- | --- | --- |
| Track selector | `>` | Currently selected track for recording [display.c L41](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L41-L41) |
| Track selector | `_` | Not selected [display.c L43](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L43-L43) |
| User note | `*` | Pattern step set by user input [display.c L53](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L53-L53) |
| Ghost note | `.` | Algorithmically generated ghost note [display.c L57](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L57-L57) |
| Fill note | `+` | Fill-in pattern note [display.c L55](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L55-L55) |
| Empty step | `_` | No note scheduled [display.c L59](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L59-L59) |

**Sources:** [display.c L39-L62](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L39-L62)

### Step Position Indicator

```
#step                  ^_______________________________
```

The `^` character marks the current playback position (0-31 steps).

### Ghost Note Calculation

Ghost notes are displayed when the following condition is met:

```
bool ghost_on =     ((float)track->ghost_notes[i].probability / 100.0f) * params->ghost_intensity >    (float)track->ghost_notes[i].rand_sample / 100.0f;
```

This compares the ghost note's probability (scaled by `ghost_intensity`) against a stored random sample to determine if the ghost note is active for the current step.

**Sources:** [display.c L48-L50](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L48-L50)

## Data Flow

The display system operates in a read-only, polling-based model where it queries the looper state on every refresh:

```mermaid
sequenceDiagram
  participant Step Timer
  participant looper_process_state()
  participant display_update_looper_status()
  participant ghost_note_parameters()
  participant UART/stdio

  Step Timer->>looper_process_state(): "Tick event"
  looper_process_state()->>display_update_looper_status(): "display_update_looper_status(ready, status, tracks, 14)"
  display_update_looper_status()->>display_update_looper_status(): "Determine state label"
  display_update_looper_status()->>UART/stdio: "Print state, BPM, grid header"
  loop ["For each track (13 down to 0)"]
    display_update_looper_status()->>ghost_note_parameters(): "ghost_note_parameters()"
    ghost_note_parameters()-->>display_update_looper_status(): "params struct"
    display_update_looper_status()->>display_update_looper_status(): "Evaluate note symbols"
    display_update_looper_status()->>UART/stdio: "Print track row"
  end
  display_update_looper_status()->>UART/stdio: "Print step indicator"
  display_update_looper_status()->>UART/stdio: "fflush(stdout)"
```

## Integration Points

### Looper Integration

The display is called from the looper's state processing function:

1. **Internal/External Clock Mode**: `display_update_looper_status()` is called within `looper_process_state()` [looper.h L85](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/looper.h#L85-L85)

The function receives the following parameters:

* `bool output_connected`: MIDI output connection status
* `const looper_status_t *looper`: Current looper state, including `bpm`, `state`, `current_track`, and `current_step` [looper.h L49-L61](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/looper.h#L49-L61)
* `const track_t *tracks`: Array of track structures, each containing `name`, `pattern`, `ghost_notes`, and `fill_pattern` [looper.h L70-L77](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/looper.h#L70-L77)
* `size_t num_tracks`: The total number of tracks [looper.h L82](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/looper.h#L82-L82)

## Output Mechanism

### UART Configuration

The display uses standard `printf()` calls that output to UART. The UART configuration is handled by the Pico SDK initialization, typically configured for 115200 baud.

**Sources:** [display.c L99-L110](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L99-L110)

### Flush Behavior

After printing all track data, the display explicitly flushes stdout to ensure immediate transmission:

```
fflush(stdout);
```

This guarantees that the complete display update is transmitted before the next step tick occurs.

**Sources:** [display.c L109](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/display.c#L109-L109)

## Performance Considerations

### Update Frequency

The display refreshes on every sequencer step. At the default 96 BPM [looper.h L12](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/looper.h#L12-L12)

:

* Step period: 156.25ms (for 16th notes, `60000 / 96 / 4 = 156.25`)
* Update rate: ~6.4 Hz
* Characters per update: ~1000 (14 tracks × ~60 chars + headers)

**Sources:** [looper.h L12](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/looper.h#L12-L12)
