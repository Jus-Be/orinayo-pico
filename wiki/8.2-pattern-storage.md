## Purpose and Scope

This document describes the flash-based persistent storage system for looper drum patterns and system preferences. The storage module ([storage.c](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c)

) provides functions to save, load, and erase data across power cycles using the Raspberry Pi Pico W's onboard flash memory. It handles both musical pattern data (32-step sequences) and operational configuration flags (e.g., Arranger mode, MIDI device enables).

**Sources:** [storage.c L1-L165](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L1-L165)

## Flash Memory Architecture

The storage system reserves a dedicated region at the end of the Pico W's flash memory. Flash operations on the RP2040 require specific alignment and the suspension of XIP (Execute In Place) access to prevent CPU stalls or crashes.

### Preference Persistence

The `storage_store_preferences` function is unique because it manually manages the CYW43 (Bluetooth/Wi-Fi) architecture. To ensure flash writing safety without using the standard `flash_safe_execute` wrapper in some contexts, it calls `cyw43_arch_deinit()` before writing and `cyw43_arch_init()` after.

**Implementation:** [storage.c L75-L104](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L75-L104)

### Pattern Persistence

Patterns are stored using the `storage_store_tracks` function. It retrieves the active track data from the looper and writes the first 4 tracks to flash.

**Process:**

1. Calls `looper_tracks_get()` to access memory-resident patterns. [storage.c L144](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L144-L144)
2. Copies `tracks[t].pattern` to a local `storage_pattern_t` buffer. [storage.c L158](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L158-L158)
3. Uses `flash_safe_execute` to perform the write, ensuring interrupts are disabled. [storage.c L162](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L162-L162)

## Safety Mechanisms

The storage implementation uses several layers of protection to ensure the integrity of both the flash hardware and the stored data.

### Flash Operation Wrapper

The `flash_bank_perform_operation` function is marked with `__no_inline_not_in_flash_func`. This attribute ensures the code is executed from RAM, which is mandatory when the flash chip is busy with erase or program cycles and cannot provide instructions to the CPU.

**Sources:** [storage.c L50](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L50-L50)

### Validation and Integrity

| Feature | Implementation | Purpose |
| --- | --- | --- |
| **Magic Header** | `MAGIC_HEADER` ("GHST") | Validates that data exists and is formatted correctly. [storage.c L17](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L17-L17) |
| **RAM Execution** | `not_in_flash_func` | Prevents CPU crashes during flash access. [storage.c L50](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L50-L50) |
| **Interrupt Safety** | `flash_safe_execute()` | Disables interrupts to prevent XIP access during writes. [storage.c L69](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L69-L69) |
| **Bluetooth Sync** | `cyw43_arch_deinit()` | Prevents Bluetooth radio activity from interfering with flash power/timing. [storage.c L98](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L98-L98) |

## API Reference

### Public Functions

#### storage_load_preferences()

Reads system flags from flash and updates global variables.

* **Returns:** `true` if magic header matches, `false` otherwise.
* **Implementation:** [storage.c L106-L128](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L106-L128)

#### storage_store_preferences()

Writes current system flags to flash.

* **Note:** Restarts the CYW43 architecture.
* **Implementation:** [storage.c L75-L104](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L75-L104)

#### storage_load_tracks()

Restores looper patterns from flash into the active `track_t` structures.

* **Implementation:** [storage.c L130-L147](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L130-L147)

#### storage_store_tracks()

Saves the first 4 looper tracks to persistent storage.

* **Implementation:** [storage.c L149-L165](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L149-L165)

#### storage_erase_tracks()

Wipes the storage sector by performing a flash erase operation.

* **Implementation:** [storage.c L67-L73](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/storage.c#L67-L73)
