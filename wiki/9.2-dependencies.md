## TinyUSB

TinyUSB is an open-source USB device stack that implements USB MIDI class support for the Orinayo system. It handles USB enumeration, descriptor parsing, and MIDI packet transmission. Orinayo uses TinyUSB in a dual-role configuration, acting as both a USB device (for MIDI output) and a USB host (for MIDI input via PIO-USB).

### TinyUSB Libraries

The system links several TinyUSB components:

1. **`tinyusb_device`** - Core USB device stack [CMakeLists.txt L70](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L70-L70)  [CMakeLists.txt L85](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L85-L85)
2. **`tinyusb_host`** - Core USB host stack [CMakeLists.txt L70](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L70-L70)  [CMakeLists.txt L85](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L85-L85)
3. **`tinyusb_board`** - Board-specific USB configuration [CMakeLists.txt L70](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L70-L70)  [CMakeLists.txt L85](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L85-L85)
4. **`pico_pio_usb`** - Library for implementing USB host functionality using the RP2040's Programmable I/O (PIO) [CMakeLists.txt L48](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L48-L48)  [CMakeLists.txt L70](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L70-L70)  [CMakeLists.txt L85](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L85-L85)
5. **`tinyusb_pico_pio_usb`** - Interface library connecting `pico_pio_usb` with TinyUSB's host stack [CMakeLists.txt L56-L62](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L56-L62)  [CMakeLists.txt L70](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L70-L70)  [CMakeLists.txt L85](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L85-L85)

The `tinyusb_pico_pio_usb` library is an INTERFACE library that defines how `pico_pio_usb` integrates with TinyUSB. It specifies source files like `dcd_pio_usb.c` and `hcd_pio_usb.c` for device and host controller drivers respectively, and includes PIO programs (`usb_tx.pio`, `usb_rx.pio`) for USB communication [CMakeLists.txt L57-L66](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L57-L66)

### USB MIDI Implementation

The system uses TinyUSB for both USB MIDI device and host functionalities:

* **Device Mode:** Implements a USB MIDI device for output. `tud_midi_n_stream_write()` is used to dispatch MIDI data over the USB endpoint.
* **Host Mode:** Implements a USB MIDI host via PIO-USB for input.

For detailed USB descriptor configuration, see [6.2 USB MIDI Configuration](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/6.2 USB MIDI Configuration)

## Bluepad32

Bluepad32 is a custom library that provides gamepad controller abstraction over Bluetooth. It translates various gamepad protocols (PlayStation, Xbox, Nintendo, etc.) into a unified input API.

### Bluepad32 Integration

**Library Location:** [CMakeLists.txt L49](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L49-L49)

### Thread-Safe Background Processing

The system defines `PICO_CYW43_ARCH_THREADSAFE_BACKGROUND` at [CMakeLists.txt L71](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L71-L71)

:

```
add_compile_definitions(orinayobt PICO_CYW43_ARCH_THREADSAFE_BACKGROUND)
```

This enables the `pico_cyw43_arch_threadsafe_background` driver, which provides:

1. **Async Context Integration** - Operations run in a background async context.
2. **SDIO Bus Arbitration** - Prevents conflicts when Bluetooth and GPIO share the bus.

## Ring Buffer Library

The `ring_buffer_lib` is a custom library providing a multi-core safe ring buffer implementation. It is used for inter-core communication and buffering data.

### Ring Buffer Integration

**Library Location:** [CMakeLists.txt L46](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L46-L46)

The library provides functions for initializing, pushing, and popping data from a ring buffer, with both safe (critical section protected) and unsafe (no critical section) variants [ring_buffer_lib/ring_buffer_lib.h L81-L193](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/ring_buffer_lib/ring_buffer_lib.h#L81-L193)

## Compiler Flags

The build system applies optimization flags to reduce binary size at [CMakeLists.txt L93-L94](https://github.com/Jus-Be/orinayo-pico/blob/6dde5a75/CMakeLists.txt#L93-L94)

:

```
set(CMAKE_C_FLAGS  "${CMAKE_C_FLAGS} -ffunction-sections -fdata-sections")set(CMAKE_CXX_FLAGS  "${CMAKE_CXX_FLAGS} -ffunction-sections -fdata-sections")
```

These flags allow the linker to perform dead code elimination, removing unused functions from the final binary.
